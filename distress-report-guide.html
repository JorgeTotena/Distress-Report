Start by placing input files in the correct folders, then compile fulfillment data into a single dataset before running any report script.

2
Compile Fulfillment Data Generates: Fulfillment_Compilation.xlsx

This step reads all files in Fulfillments/, adds Month, Marketing_Channel, and Source_File columns, and stacks them into one master dataset (77,000 rows, 37,338 unique FOLIOs).

Run in terminal

Report Generation — Steps 3, 4 & 5
With fulfillment data compiled, build the domain cache once, then run the main report script to produce the final Excel output.
3
Build the Domain Cache One-time setup

The domain data ships as two large Excel parts. compile_domain.py merges them and saves a fast-loading domain.parquet file. Only run this once — or when you receive updated domain files.

python compile_domain.py # Output: Domain Full Data/domain.parquet (536,941 rows) # DEFAULT RISK is embedded directly in this parquet — no separate file needed.
4
Optional — Column C Only Report Quick validation

Run build_reports.py to generate a Column C-only report before the full run. Useful for validating score bucketing, action plans, distress counts, and marketing count logic against the expected result template.

python build_reports.py # Outputs: Expected_Result_Max.xlsx, Expected_Result_Average.xlsx

Claude Code prompt: "Run build_reports.py and compare the Likely Deal Score distribution in the output against the values in Expected Result/Atlas - Expected Result.csv."

5
Run the Full Report Main output

This is the primary script. It loads all data sources, applies MAX logic across all columns, handles Deals/Leads/Appointments filtering by CLIENT_START_DATE, writes Excel formulas for concentration columns, and saves the final report.

python build_domain_report.py # Output: Expected_Result_Max_BCDFG_InFulfillment.xlsx
Before running — set client start date
# Line ~36 of build_domain_report.py CLIENT_START_DATE = pd.to_datetime("2021-10-26") # Change this for each new client

Reference
All output files, key business logic decisions, and guidance for running the report for a new client are documented below.
Output Files
FilePurpose
Expected_Result_Max_BCDFG
_InFulfillment.xlsx		Primary output. Full report with all columns B–I.
Fulfillment_Compilation.xlsxAll fulfillment files stacked (77k rows). Used as input by main script.
Population_For_Calculation.xlsxAudit file — one row per unique property with all calculated values.
Expected_Result_Max.xlsxColumn C only (MAX logic). For validation.
Expected_Result_Average.xlsxColumn C only (AVERAGE logic). For validation.
Key Business Logic
  • Marketing Count: max(total DM, total SMS) per property across all months — not the sum. A property with 7 SMS + 4 DM = count of 7.
  • Action Plan: Minimum days (30 < 60 < 90) across all months — the most aggressive plan wins.
  • Leads filter: LEAD DATE ≥ CLIENT_START_DATE. Appointments filtered by APPOINTMENT DATE, then reclassified as Leads.
  • Contracts: Excluded — no reliable date column available.
  • Owner Occupied: Derived from ABSENTEE = 0 in domain via FOLIO join. Not available directly from fulfillment distress columns.
Running for a New Client
  1. 1 Replace Deals and Leads Freedom.xlsx in Documents/ with the new client's file. Confirm column names match.
  2. 2 Update DEALS_PATH and CLIENT_START_DATE at the top of build_domain_report.py.
  3. 3 Drop the new fulfillment files in Fulfillments/. Re-run build_population.py to recompile.
  4. 4 Run build_domain_report.py. Only rerun compile_domain.py if you received new domain data files.
