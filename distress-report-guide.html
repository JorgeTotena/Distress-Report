What the Report Produces

The output is a single Excel file named {CLIENT_NAME} - Distress Report - YYYY-MM.xlsx (e.g. SBD - Distress Report - 2026-04.xlsx) with one row per distress category and these columns:

ColDescriptionSource
BTotal PropertiesDomain (BUYBOX > 0)

DSold since Aug 1, 2025Domain last sale date
EMarket DealsProperties bought at a significant discount, resold for profit in a short period — investor deals ∩ fulfillment-sold
FClients LeadLeads + Appointments + Dead Leads + Contracts
GClient DealsFulfillment-matched deals
H–KConcentration %Excel formulas (D/C, E/C, G/C, F/C)
Project Folder Structure
# Working directory layout Distress Report/ ├── build_domain_report.py # MAIN script ├── compile_domain.py # Re-run when domain files change ├── Fulfillments/ # Monthly source xlsx files ├── Documents/ # Deals & Leads file ├── Domain Full Data/ # domain.parquet + source xlsx ├── Market Deals/ # SBD Market Deals.xlsx └── Expected Result/ # Template CSV

Fulfillment Window Rule: Standard is 6 months ending 3 months before the analysis date. Formula: most recent month = analysis month − 3, then include the 5 months before that (6 total). Example: analysis in April 2026 → use August 2025 through January 2026. Fallback: if 6 months of files are not available, use 3 months; if fewer than 3 are available, use whatever is there. The window and Column D sold cutoff are computed dynamically at runtime — nothing is hardcoded.

Before you begin: You need Python 3.10+ with pandas, openpyxl, numpy, and pyarrow installed. Run pip install pandas openpyxl numpy pyarrow if not already set up. Claude Code must be open in the project root.

2
Compile the Domain Cache Re-run when domain files change

The domain data ships as two large Excel parts. compile_domain.py merges them and saves a fast-loading domain.parquet file. Only run this when you receive updated domain xlsx files — the parquet persists between report runs.

Run in terminal
python compile_domain.py
Or prompt Claude Code
"Run compile_domain.py and confirm how many rows and unique FOLIOs are in the new parquet."
Expected Console Output
# Successful run looks like: Found 2 file(s): ['...part_1.xlsx', '...part_2.xlsx'] Reading part 1: ... 500,000 rows Reading part 2: ... 216,089 rows Combined: 716,089 rows, 709,929 unique FOLIOs Saved: Domain Full Data/domain.parquet

Note on DEFAULT RISK: Some domain exports include a DEFAULT RISK column; others do not. The script detects this automatically. If the column is absent, Default Risk counts will be 0 and a warning is printed — no crash, no manual intervention needed.

Report Generation — Steps 3, 4 & 5
With files organized and the domain cache ready, run the main script. It handles fulfillment compilation, all column calculations, and Excel formatting in a single pass.
3
Run the Full Report Main output

This is the only script you need to run for a standard report. It compiles fulfillment files, loads the domain parquet, computes all columns (B–G), writes concentration formulas (H–K), and saves the final Excel output. The fulfillment compilation audit file is also saved automatically each run.

Run in terminal
python build_domain_report.py # Output: {CLIENT_NAME} - Distress Report - YYYY-MM.xlsx # e.g. SBD - Distress Report - 2026-04.xlsx # Audit: Fulfillment_Compilation.xlsx
Or prompt Claude Code
"Run build_domain_report.py and give me a summary of the output — total properties, fulfillment count, sold, market deals, leads, and deals."
Before running — client settings
# Top of build_domain_report.py — update for each client CLIENT_NAME = "SBD" # Used in output filename: {CLIENT_NAME} - Distress Report - YYYY-MM.xlsx CLIENT_START_DATE = pd.to_datetime("2023-09-26") # SBD start date with 8020REI MARKET_PATH = BASE / "Market Deals" / "SBD Market Deals.xlsx" # Update filename for each new client

The analysis window is fully automatic. The script computes the 6-month window from today's date at runtime — no date needs to be changed between monthly runs.

Output Files
FilePurpose

- YYYY-MM.xlsx		Primary output. Full report with all columns B–K. Filename uses CLIENT_NAME and run month (e.g. SBD - Distress Report - 2026-04.xlsx).
Fulfillment_Compilation.xlsxAudit file — all fulfillment rows stacked (668k rows for SBD). Written automatically each run.

Expected_Result_Average.xlsxColumn C only (AVERAGE logic). For validation.
Key Business Logic
  • Marketing Count: max(total DM, total SMS) per property across all months — not the sum. A property with 7 SMS + 4 DM = count of 7.
  • Action Plan: Minimum days (30 < 60 < 90) across all months — the most aggressive plan wins.
  • Leads filter: Four statuses count as a Lead in Column F: Lead, Appointment, Dead Lead, and Contract. All are reclassified as Lead before counting. Date columns: Lead/Dead Lead/Contract → LEAD DATE; Appointment → APPOINTMENT DATE; Deal → LAST SALE DATE. Blank dates for lead-type statuses are included — the date is unknown but the record cannot be ruled out. Deals must have a valid date.
  • Market Deals (Column E): Properties believed to have been bought at a significant discount — typically bought and resold for a profit in a short period of time. Computed as the overlap between the Market Deals file and the fulfillment-sold subset, joined via PropertyID = PROPERTY ID (BUYBOX). Scores and distress are taken from pre-sale fulfillment values (same as Column D).
  • Owner Occupied: Derived from ABSENTEE = 0 in domain via FOLIO join. Not available directly from fulfillment distress columns — cross-referenced for all columns.
Running for a New Client
  1. 1 Drop the client's Deals & Leads .xlsx into Documents/. The script auto-discovers any file whose name contains leads or deals.
  2. 2 Drop the client's Market Deals .xlsx into Market Deals/ and update MARKET_PATH at the top of build_domain_report.py.
  3. 3 Update CLIENT_NAME and CLIENT_START_DATE at the top of build_domain_report.py. CLIENT_NAME drives the output filename; CLIENT_START_DATE filters leads and deals.
  4. 4 Drop the new fulfillment files in Fulfillments/. If new domain data was also provided, replace the xlsx parts in Domain Full Data/ and re-run compile_domain.py.
  5. 5 Run build_domain_report.py.
